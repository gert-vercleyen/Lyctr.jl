# Lyctr

[![Build Status](https://github.com/gert-vercleyen/Lyctr.jl/actions/workflows/CI.yml/badge.svg?branch=main)](https://github.com/gert-vercleyen/Lyctr.jl/actions/workflows/CI.yml?query=branch%3Amain)

Lyctr is a julia package that provides tools for working with skeletal fusion categories. The name derives from the word `Lyctor', an immortal necromancer in the [Locked Tomb Series](https://en.wikipedia.org/wiki/The_Locked_Tomb) (a must read for anyone using skeletal categories ;))

Lyctr is developed by the developer of Anyonica but is written in julia and uses [Oscar](https://www.oscar-system.org/) for its symbolic computations and exact representations of numbers. The goal is to link it intimitely with a database that fuels the new version of the [anyonwiki](https://anyonwiki.org/). In particular, both the wiki and Lyctr will draw from the same open-source database (still in the making) and Lyctr will be the package that updates the database. 

# Important!
* [anyonwiki](https://anyonwiki.org/) is under active development and might contain serious errors. 
* Only the code on the `main` branch is production ready and. It will take some time before the whole of Anyonica is ported to Lyctr.
